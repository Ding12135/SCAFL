# 实现状态：论文对齐审查（A/B/C）

本文档整理当前 SC-AFL P2 形态实现与论文对象的对齐程度，**不涉及代码修改**。符号与字段细节见 [`paper_symbol_mapping.md`](paper_symbol_mapping.md)。

---

## 最终结论

**当前实现应表述为「论文启发实现 / engineering approximation」，不应表述为「论文复现」。**

依据：P2 **目标式的代数结构**与文档中的公式一致，且 **\(V\)** 在目标中的角色可配置、可核对；但 **\(Q_k,\tau_k,\tau_{\max},D_t\)** 的物理语义、**可行决策族（前缀启发式）**、**队列更新时刻** 等与论文中的完整数学对象/优化问题**未证明等价**，故不满足「在同一模型上复现论文结果」的严格含义。

---

## 分类说明

| 类别 | 含义 |
|------|------|
| **A** | 与论文在该对象上的定义/角色**可称严格一致**（在已声明的数学结构内）。 |
| **B** | **形式或结构**对齐，**物理含义或定义域**仍为工程近似或与论文未证明等价。 |
| **C** | **仍未**与论文中的优化对象或可行集对齐；若宣称复现**必须先改**。 |

## 符号与模块：A/B/C 一览

| 对象 | 类别 | 一句话 |
|------|------|--------|
| **\(V\)** | **A** | 作为 P2 时间项系数进入目标，由 `policy.V` 配置，无隐藏缩放。 |
| **\(Q_k(t)\)** | **B** | `virtual_queue` + 事件边界递推；与论文 Lyapunov 中 \(Q_k\) 的完整递推/定义域未必一致。 |
| **\(\beta_k^t\)** | **B** | 实现为 **client-level**（多 pending 合并）；若论文为 per-update 或连续过程则存在语义差。 |
| **\(\tau_k(t)\)** | **B** | `staleness` + client 上取 max；非论文连续时间 \(\tau_k(t)\) 的唯一标准定义。 |
| **\(\tau_{\max}\)** | **B** | `tau_max_t` / 策略回退 `target_size`；与论文中固定系统参数的关系未形式化。 |
| **\(D_t\)** | **B** | `max(d_k)`，`d_k` = 计算+上传延迟和；与论文「轮次墙钟跨度」类定义未必同一物理量。 |
| **P2 objective** | **B** | 公式逐项实现，但输入变量整体为 B 类近似，故整体非「同一物理系统上的同一目标」。 |
| **Prefix selection rule** | **C** | 仅在按 \(d_k\) 排序后的**前缀族**上 argmin，非论文可行集上的全局最优。 |
| **Aggregated subset \(M_t\)** | **B** | 对应 buffer 快照内选中行；论文 \(M_t\) 若指系统级全集则定义域不同。 |
| **Queue update timing boundary** | **B** | 聚合事件后更新 \(Q_k\)；与论文离散槽/连续时间索引未建立一一对应证明。 |

## 若要达到「论文复现」，下一步必须修改什么（简短列表）

以下为**方向性要求**，不展开实现细节：

1. **Prefix rule（C）**：在论文给定的**可行决策集**上实现与论文一致的优化（完整枚举、解析解或可证等价算法），或**正式修改论文问题陈述**为「受限前缀族上的启发式」并放弃「复现原定理」。
2. **\(\tau_k, Q_k, \tau_{\max}, D_t\)（B）**：采用论文中的**显式定义**（时间轴、到达过程、\(Q_k\) 递推式），替换或证明与当前 staleness / virtual_queue / `max(d_k)` / 控制器阈值的等价性。
3. **\(\beta_k\)（B）**：与论文对齐为 **per-update** 或 **client-level** 之一，并同步修改 P2 中聚合方式与日志语义。
4. **\(M_t\) 与 queue timing（B）**：统一 **全局时间索引** 与 **更新触发条件**，使 \(M_t\) 的定义域、\(Q_k\) 的更新时刻与论文模型一致。
5. **验证**：在固定随机种子与论文相同实验设定下，对**关键量**（目标值、队列轨迹、服务决策）与论文报告做**可重复对照**；若论文给出定理条件（如 \(V\) 的界），增加**可行性检查**。

完成以上并使审查方接受后，方可考虑使用「论文复现」表述。
